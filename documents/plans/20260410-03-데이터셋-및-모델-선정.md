### 선정 기준: 왜 이 5개인가

#### 기준 1. 프로젝트 목적에 부합하는 모델 다양성

> 우리의 목적: "DSC 점수가 하락하면 모델 성능도 하락한다"를 입증

이 주장이 **특정 모델에서만 성립하면 가치가 없다.** 다양한 학습 원리를 가진 모델에서 공통적으로 성립해야 설득력이 있다. 따라서 학습 원리가 서로 다른 5개 계열을 선택:

| 계열 | 학습 원리 | 대표 모델 |
|---|---|---|
| 선형 모델 | 피처의 선형 결합으로 결정 경계 학습 | LogisticRegression |
| 거리/커널 기반 | 데이터 포인트 간 거리/유사도로 판단 | SVC |
| 배깅 앙상블 | 랜덤 서브샘플링 + 다수결 투표 | RandomForest |
| 부스팅 앙상블 | 이전 모델의 오류를 순차 보정 | XGBoost |
| 신경망 | 가중치 역전파로 비선형 함수 학습 | MLP |

#### 기준 2. 데이터 품질에 대한 민감도 스펙트럼

각 모델은 데이터 품질 문제에 대한 **내성(robustness)이 다르다.** 이 차이가 우리 실험의 핵심 인사이트를 만든다:

| 모델 | 결측치 | 노이즈 | 중복 | 클래스 불균형 | 이상치 | 종합 민감도 |
|---|---|---|---|---|---|---|
| **LogisticRegression** | 처리 불가 (에러) | 높음 — 선형 가정이 깨짐 | 낮음 | 높음 — 다수 클래스 편향 | 높음 — 회귀선 왜곡 | **매우 민감** |
| **SVC** | 처리 불가 (에러) | 높음 — 서포트 벡터 오염 | 중간 | 높음 — 결정 경계 편향 | 높음 — 마진 왜곡 | **매우 민감** |
| **RandomForest** | 일부 처리 가능 | 낮음 — 배깅이 평균화 | 낮음 | 중간 — 트리별 분산 | 낮음 — 분할 기준 강건 | **강건** |
| **XGBoost** | 자체 처리 가능 | 낮음 — 정규화 내장 | 낮음 | 중간 — scale_pos_weight | 중간 — 부스팅 누적 | **강건** |
| **MLP** | 처리 불가 (에러) | 중간 — 과적합 경향 | 중간 — 가중치 편향 | 중간 — 출력층 편향 | 중간 — 활성화 포화 | **중간** |

이 스펙트럼이 중요한 이유:

```
실험 결과 예상 시나리오:

DSC 90→70 (20점 하락) 일 때:
  LogisticRegression  F1: 0.85 → 0.68  (Δ -0.17)  ← 민감
  SVC                 F1: 0.83 → 0.65  (Δ -0.18)  ← 민감
  MLP                 F1: 0.84 → 0.72  (Δ -0.12)  ← 중간
  RandomForest        F1: 0.86 → 0.79  (Δ -0.07)  ← 강건
  XGBoost             F1: 0.87 → 0.81  (Δ -0.06)  ← 강건

→ 모든 모델에서 성능이 하락 = DSC 유효성 입증
→ 하락 폭이 다름 = "어떤 모델을 쓸 때 데이터 품질이 더 중요한가" 인사이트
```

#### 기준 3. 각 모델의 구체적 선정 이유

**LogisticRegression — "바닥 기준선"**
- 가장 단순한 모델이자 ML의 출발점
- 데이터 품질 문제가 곧바로 성능에 반영됨 (버퍼 없음)
- 오염 효과를 가장 명확하게 보여주는 "리트머스 시험지"
- 학술 논문에서 baseline으로 거의 항상 포함

**SVC (Linear) — "거리 민감 모델"**
- 결정 경계가 소수의 서포트 벡터에 의존 → 이상치/노이즈에 극도로 민감
- 스케일링 필수 → Validity(형식 오류) 오염 시 성능 급락 예상
- LogisticRegression과 같은 선형이지만 학습 원리가 다름 (마진 최대화 vs 확률 최대화)

**RandomForestClassifier — "강건한 모델의 대표"**
- 배깅 + 랜덤 피처 선택 → 노이즈/이상치 자연 필터링
- 결측치도 일부 처리 가능 (구현에 따라)
- "데이터 품질이 나빠도 어느 정도 버틴다"는 것을 보여줌
- → DSC가 낮아도 RF 성능은 유지 = "모델에 따라 품질 기준이 달라야 한다"

**XGBClassifier — "실무 표준"**
- Kaggle/산업에서 가장 널리 쓰이는 tabular 모델
- 결측치 자체 처리 가능 (`missing` 파라미터) → Completeness 오염에 대한 독특한 반응
- L1/L2 정규화 내장 → 노이즈 내성
- RF와 비교: 같은 트리 기반이지만 배깅 vs 부스팅의 차이가 오염 반응에 어떻게 나타나는지

**MLPClassifier — "신경망 계열 대표"**
- 딥러닝의 기본 구조 (5층 퍼셉트론)
- 오버피팅 경향 → 중복/노이즈 데이터를 그대로 학습할 위험
- 스케일링 필수 → Validity 오염에 민감
- 비선형 결정 경계 → 선형 모델과 다른 오염 반응 패턴 기대

#### 기준 4. DQ4AI 논문과의 비교 가능성

DQ4AI 논문이 11개 모델로 실험한 결과가 이미 있으므로, 우리가 같은 계열 모델을 쓰면:
- 우리 실험 결과와 DQ4AI 결과를 **교차 검증** 가능
- "DQ4AI가 보여준 오염-성능 관계가 DSC 점수로도 포착된다"는 주장 가능
- 특히 LogisticRegression, SVC, MLP, GradientBoosting은 DQ4AI와 **직접 대응**

#### 기준 5. 실용성

- 5개 모두 **sklearn 기반** → 동일 API (`fit`, `predict`, `predict_proba`)
- GPU 불필요 → Colab 무료 티어로 충분
- 440회 실험이 ~3.5시간 내에 완료
- 하이퍼파라미터 튜닝 불필요 (고정값 사용 → 공정한 비교)

### 하이퍼파라미터 (DQ4AI 기준 고정)

```python
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier
from sklearn.svm import SVC
from sklearn.neural_network import MLPClassifier
from xgboost import XGBClassifier

MODELS = {
    "LogisticRegression": LogisticRegression(
        solver='lbfgs', max_iter=2000, random_state=42, n_jobs=-1
    ),
    "RandomForest": RandomForestClassifier(
        n_estimators=100, random_state=42, n_jobs=-1
    ),
    "XGBoost": XGBClassifier(
        n_estimators=100, random_state=42, n_jobs=-1,
        use_label_encoder=False, eval_metric='mlogloss'
    ),
    "SVC": SVC(
        kernel='linear', random_state=42, probability=True
    ),
    "MLP": MLPClassifier(
        hidden_layer_sizes=(100, 100, 100, 100, 100),
        random_state=42, max_iter=1000
    ),
}
```

> 하이퍼파라미터 튜닝은 하지 않음. 모든 실험에서 동일 파라미터 사용 → 공정한 비교
