## 0. 배경 — 메타 검증에서 드러난 결함

진단·개선 보고서(v3.2)가 데이터 품질 차원의 결함(P1~P7)은 해결했으나, **실험·정의·통계적 추론 자체의 논리 결함**은 미해결로 남아 있다. 메타 검증 결과 다음 10개 결함이 식별되었다.

| ID | 결함 | 심각도 |
|---|---|---|
| F1 | v3.2 가중치를 ML 결과 보고 조정 → 순환 논증 | **치명** |
| F2 | r=0.42 ⇒ r²=0.18 → "예측" 주장 약함 | **심각** |
| F3 | value_accuracy(reference 의존)가 30% 차지 → "품질 점수" 정의 모순 | **심각** |
| F4 | DSC = Σ wᵢ sᵢ 가법성 가정 무근거 | 중간 |
| F5 | 5종 합성 polluter → "데이터 품질 전반" 일반화 비약 | 중간 |
| F6 | 모델별 r 격차(0.31~0.55) → 모델 무관 지표 정의 모순 | 약함 |
| F7 | 데이터셋 confounding (Telco r=0.15, 비유의) | 중간 |
| F8 | baseline = "깨끗한 데이터" 가정 무근거 | 약함 |
| F9 | 등급 임계값 90/75/60의 자의성 | 약함 |
| F10 | F1 macro 단일 의존 | 약함 |

**제약 조건**: DSC는 **반드시 "데이터 품질 점수"** 의미를 유지해야 함. 정의를 "drift 측정기"로 바꿀 수 없음.

## 2. Phase별 상세

### Phase 1 — F3: value_accuracy 교체

**목표**: 가중치 0.30짜리 reference 의존 지표를 reference-free 절대 품질 지표로 교체.

**신설 지표 2개**:

| 지표 | 계산 | ML 인과 | 가중치 |
|---|---|---|---|
| `label_consistency` | k=5 KNN. 각 샘플의 이웃 라벨 중 자기 라벨과 같은 비율의 평균 (수치형 컬럼만 사용, sklearn `KNeighborsClassifier`) | 라벨 노이즈·결정경계 모호도. ML 학습 가능성 직접 측정 | 0.20 |
| `feature_informativeness` | `sklearn.feature_selection.mutual_info_classif` 평균 | 피처가 라벨에 주는 정보량 | 0.10 |

**신 가중치 (합 = 1.00)**:
```
completeness:           0.20
uniqueness:             0.15
validity:               0.05
consistency:            0.10
outlier_ratio:          0.05
class_balance:          0.10
feature_correlation:    0.05
label_consistency:      0.20  ← 신설
feature_informativeness:0.10  ← 신설
```

⚠️ value_accuracy 제거. reference_df 인자는 outlier_ratio용으로만 유지.

**작업 산출물**:
- `notebooks/_dev/apply_dsc_engine_v4.py` — 노트북 01·02 패치 스크립트
- `notebooks/_dev/verify_v4_metrics.py` — 단위 검증
- `documents/decisions/ADR-009-DSC엔진v4-절대품질지표교체.md`

**검증 기준**:
- baseline에서 두 신설 지표 모두 0.7 이상 (정상 데이터에서 정보 풍부)
- feature_accuracy 0.75 폴루션에서 label_consistency 베이스라인 대비 −0.10 이상 하락
- completeness 0.75 폴루션에서 두 지표 모두 적절히 하락
- uniqueness 0.75 폴루션에서 두 지표 거의 변화 없음 (false positive 검증)

### Phase 2 — F1: Polluter Hold-out 검증

**목표**: v3.2/v4.0 가중치가 ML 결과를 보고 조정된 fitting 결과인지, 아니면 진짜 일반화되는 신호인지 검증.

**방법**:
1. 5개 polluter (completeness, uniqueness, consistent_repr, class_balance, feature_accuracy) 각각을 hold-out으로 지정.
2. 각 hold-out 슬라이스에서만 Pearson r 계산.
3. 모든 hold-out에서 r > 0.3 + p < 0.05 면 "fitting 아닌 진짜 입증".

**작업 산출물**:
- `notebooks/04_scoreboard.ipynb`에 새 셀 추가 (또는 `_dev/holdout_analysis.py`로 분리)

**검증 기준**:
- 5개 hold-out 평균 r ≥ 0.30
- 최소 4/5 polluter에서 p < 0.05

### Phase 3 — F2/F4: 주장 강도 + 가법성 한계

**F2 처리**:
- 04 로그·발표문에서 "예측" → "유의 상관"으로 정정
- r²(=0.18 추정) 명시 보고
- 비선형 회귀 R² 추가: 8개 metric을 입력으로 RandomForestRegressor가 F1 예측. R²이 0.5+ 나오면 "선형 결합의 한계로 r²=0.18, 비선형 결합 시 정보량 충분" 강력 방어.

**F4 처리**:
- min-aggregation 비교: `DSC_min = min(metrics) * 100`
- 두 모델의 r 비교 후 결과 보고
- 한계 명시: "현재 DSC는 차원 간 독립을 가정한 선형 근사"

## 3. 검증 원칙 (CLAUDE.md 준수)

1. **모든 코드 수정은 실제 train_polluted/ 데이터로 단위 검증** — 구문 검사만으로 끝내지 않음.
2. **Split-first 원칙 유지** — label_consistency, feature_informativeness 모두 train만 사용 (test 누수 없음).
3. **사후 필터링 금지** — DSC 점수 결과를 보고 가중치를 또 조정하지 않음. v4 가중치는 본 마스터플랜에서 사전 등록(pre-register)한 값을 그대로 사용.

## 4. 사전 등록(Pre-registration) — Phase 1 가중치 고정

본 문서를 작성하는 시점(2026-04-27)에 **다음 가중치를 미리 확정**한다. 이후 시뮬레이션 결과를 보고 가중치를 수정하지 않는다 (F1 순환 논증 회피의 핵심).

```
Default profile:
  completeness:           0.20
  uniqueness:             0.15
  validity:               0.05
  consistency:            0.10
  outlier_ratio:          0.05
  class_balance:          0.10
  feature_correlation:    0.05
  label_consistency:      0.20
  feature_informativeness:0.10
```

근거:
- completeness 0.20: ISO/IEC 25012 1순위 차원, v3.2 그대로
- uniqueness 0.15: v3.2 그대로 (P5 한계로 더 낮추는 안도 있으나 일관성 위해 유지)
- validity 0.05: 본 실험에서 변별력 없으나 외부 일반화 위해 유지
- consistency 0.10: v3.2 그대로
- outlier_ratio 0.05: v3.2 그대로
- class_balance 0.10: v3.1에서 강화한 값 유지
- feature_correlation 0.05: v3.2 그대로 (데이터셋 상수, 가중치 작게)
- **label_consistency 0.20**: ML 인과 가장 직접, value_accuracy 0.30의 2/3 이전
- **feature_informativeness 0.10**: 보조 신호, 0.30의 1/3 이전
