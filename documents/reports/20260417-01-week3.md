## 0. 연구 배경 및 핵심 용어

### 0.1 왜 이 연구를 하는가
머신러닝 프로젝트에서 "데이터가 중요하다"는 말은 반복되지만, **실제로 데이터 품질을 수치로 정량화하여 모델 성능을 예측할 수 있는지**는 아직 충분히 검증되지 않았다. 본 프로젝트는 데이터 품질 점수 체계(DSC)를 정의하고, **"DSC 점수가 높을수록 모델 성능이 좋다"는 상관관계를 실험적으로 입증**하는 것을 목표로 한다.

### 0.2 핵심 가설
```
깨끗한 데이터 → DSC 점수 높음 → 모델 성능 좋음
오염된 데이터 → DSC 점수 낮음 → 모델 성능 나쁨
∴ DSC 점수 변화량과 모델 성능 변화량 간에 유의한 상관관계가 있다.
```

### 0.3 용어 정의

| 용어 | 설명 |
|---|---|
| **DSC (Data Score Card)** | 데이터 품질을 7개 지표의 가중 평균으로 0~100점으로 계산한 체계. 90+ A / 75~89 B / 60~74 C / 60 미만 D 등급으로 분류 |
| **DQ4AI** | 독일 HPI 연구실의 오픈소스 도구. 데이터에 인위적 오염(결측, 중복, 값 왜곡 등)을 주입하여 품질 저하를 시뮬레이션 |
| **Polluter** | DQ4AI가 제공하는 오염 주입 모듈. 본 실험에서 5종 사용 (완전성/유일성/정확성/일관성/클래스균형) |
| **오염 강도 (Pollution Level)** | 전체 데이터 중 오염이 적용된 비율 (0% = 원본, 75% = 4분의 3이 오염) |

### 0.4 DSC 7개 품질 지표

| 지표 | 측정 대상 | 가중치 |
|---|---|---|
| Completeness | 결측치가 없는 비율 | 0.25 |
| Uniqueness | 중복이 없는 행의 비율 | 0.20 |
| Validity | 타입/형식이 올바른 비율 | 0.20 |
| Consistency | 범주값 표기가 일관된 비율 | 0.15 |
| Outlier Ratio | IQR 기준 이상치가 아닌 비율 | 0.10 |
| Class Balance | 타겟 클래스 분포의 균형도 | 0.05 |
| Feature Correlation | 고상관(>0.95) 피처 쌍이 없는 비율 | 0.05 |

`DSC Score = Σ(지표 × 가중치) × 100`

### 0.5 실험 규모
- **데이터셋 3종**: TelcoCustomerChurn(이진분류, 7,043건) / SouthGermanCredit(이진분류, 1,000건) / Letter Recognition(26-클래스, 20,000건)
- **모델 5종**: LogisticRegression, SVC, RandomForest, XGBoost, MLP
- **전체 학습 횟수**: 3 데이터셋 × 5 polluter × 4 강도 × 5 모델 + 베이스라인 = **약 315회**

### 1.2 환경 구축 및 베이스라인 측정 (04-11)
- DQ4AI 오픈소스 클론 및 데이터셋 3종 로드 완료
- **DSC 스코어링 엔진 v1 구현**: 7개 품질 지표를 입력 데이터프레임에서 자동 계산
- 오염 없는 원본 데이터의 베이스라인 측정 완료

| 데이터셋 | 베이스라인 DSC | 등급 | 최고 성능 모델의 F1 |
|---|---|---|---|
| TelcoCustomerChurn | 97.65 | A | LR 0.739 |
| SouthGermanCredit | 97.45 | A | XGBoost 0.742 |
| Letter | 98.11 | A | XGBoost 0.959 |

→ 원본 데이터가 모두 A등급임을 확인, "깨끗한 데이터 = 높은 DSC" 기준선 확립

### 1.3 오염 실행 및 DSC 엔진 결함 발견 (04-12)
5종의 오염을 3종 데이터셋에 4단계 강도(10/25/50/75%)로 주입한 결과, **DSC 점수가 오염 강도에 충분히 반응하지 않는 현상**을 발견했다.

**구체적 증상** (TelcoCustomerChurn 기준):

| 오염 유형 | 0% → 75%에서 DSC 하락폭 | 평가 |
|---|---|---|
| Completeness (결측 주입) | 97.65 → 76.34 (-21.3점) | 정상 반응 |
| Uniqueness (중복 주입) | 97.65 → 82.65 (-15.0점) | 정상 반응 |
| Consistency (표기 왜곡) | 변화 거의 없음 | 지표 로직 결함 |
| Feature Accuracy (값 왜곡) | 97.65 → 97.61 (-0.04점) | 감지 불가 |
| Class Balance (클래스 불균형) | 97.65 → 96.25 (-1.4점) | 민감도 부족 |

**원인 분석**:
- Completeness 지표가 `NaN`만 카운트 → 오염 도구는 `-1`, `"empty"` 같은 placeholder를 삽입하므로 감지 실패
- Consistency 지표가 유니크값 개수만 비교 → 오염의 "양"을 반영 못함
- Class Balance는 엔트로피 기반으로 이진분류에서 민감도 부족

### 1.4 DSC 엔진 v2 재설계 (04-12)
문제가 된 3개 지표를 재설계하여 오염 강도에 비례해 점수가 변하도록 수정했다.

| 지표 | 이전 로직 | 개선 로직 |
|---|---|---|
| Completeness | `isnull()` 결과만 카운트 | 사용자 정의 placeholder 값도 결측으로 카운트 |
| Consistency | 범주값 유니크 개수 비교 | 오염 표식(접미사)이 붙은 **행의 비율** 직접 측정 |
| Class Balance | 엔트로피 / 최대 엔트로피 | 최소 클래스 비율 / 이상적 균등 비율 |

추가로 4개 노트북 모두에 **실행 로그 자동 생성 셀**을 추가하여 재현성 및 결과 추적성을 확보했다.

### 1.5 전체 파이프라인 재실행 및 통계 분석 (04-12 ~ 04-13)
개선된 엔진으로 전체 실험을 재실행하고 통계 분석을 수행했다.

- 생성된 데이터: **DSC 점수 59건 · 모델 성능 295건**
- 상관관계 분석: **Pearson r = 0.085 (p = 0.147)** — 통계적으로 유의하지 않음
- 등급별 비교: A등급 평균 F1 = 0.698 (n=210), B등급 평균 F1 = 0.682 (n=85), ANOVA p=0.375로 차이 비유의
- 추세 자체는 "DSC 10점 하락 시 F1 약 0.019 하락"으로 **방향성은 가설과 일치**하지만 통계적 유의성은 확보되지 않음
- 시각화 차트 5종 생성 (산점도·강도별 변화·민감도 히트맵·등급별 박스플롯·레이더)

## 2. 현재 결과 해석 및 문제점

### 2.1 결과의 의미
현재 단계의 결과는 가설의 **반증이 아니라, 실험 설계의 해상도가 부족한 상태**로 해석된다. 구체적으로:

1. **DSC 점수 분포가 너무 좁다**: 오염 75%를 가해도 B등급까지만 내려가고 **C·D 등급이 전혀 생성되지 않음**. 품질이 "보통~좋음" 구간에만 몰려 있어 상관관계를 검출할 변화 폭(dynamic range)이 부족하다.
2. **여전히 감지되지 않는 오염**: DSC 엔진 v2에서도 Feature Accuracy 오염(숫자값 왜곡)과 Class Balance 오염이 거의 감지되지 않는다. 타입과 형식은 유지되지만 값 자체가 왜곡되는 오염을 탐지할 지표가 현 체계에 없다.
3. **결과적으로 "DSC ↔ 성능" 핵심 주장은 아직 입증 전 단계**이다.

### 2.2 이번 단계에서 얻은 학술적 발견
실험이 바로 성공하지는 않았지만, **"어떤 종류의 데이터 오염은 현재 DSC 체계로 감지할 수 없다"는 사실 자체가 유의미한 발견**이다. 이는 DSC라는 데이터 품질 평가 체계를 실용화하려면 어떤 지표가 추가로 필요한지에 대한 구체적 단서를 제공한다.

## 3. 차기 계획

### 3.1 DSC 엔진 v3 개선 (최우선)
- **값 분포 변화를 감지하는 지표 신설**: Kolmogorov-Smirnov 검정, Wasserstein 거리 등 분포 거리 기반 지표를 도입하여 Feature Accuracy 오염 탐지
- **Class Balance 가중치 재조정**: 0.05 → 0.10~0.15 상향으로 이진분류에서의 민감도 개선
- **Validity 지표 강화**: 타입·형식 외에 값의 범위 및 도메인 제약까지 포함

### 3.2 오염 강도 확장
- 현재 최대 75% → **85%, 95%, 100%** 강도 추가하여 C·D 등급 데이터 확보
- 여러 오염을 동시에 적용하는 **복합 오염 실험**으로 극단 케이스 생성

### 3.3 재실험 및 통계 검증
- 개선된 엔진으로 전체 파이프라인 재실행 (약 500회 학습 예상)
- **검증 목표: Pearson r ≥ 0.5, p < 0.01, 등급 간 ANOVA 유의**
- 목표 미달 시 실패 요인을 구체적으로 분석하여 논문 한계점 섹션에 기술

### 3.4 발표 자료 준비
- 차트 5종 가독성 개선 (축 단위, 범례, 색상)
- 핵심 주장을 뒷받침하는 수치 문장화
- DSC 체계의 한계 및 향후 연구 방향 정리
