# DSC 점수가 ML 성능을 예측하는가?

**2026-04-27**

> 가설: 깨끗한 데이터(DSC↑)일수록 ML 성능(F1↑)이 높다 — 통계적으로 입증할 수 있는가?

## 결정적 결함 P1·P2 + 메타 결함 10개

### 데이터 차원 결함 (P1·P2)
- **P1** outlier_ratio가 가우시안 노이즈에 **역반응** — 자기참조 함정
- **P2** validity가 타입만 검사 → 가우시안 노이즈 무감지

### 메타 논리 결함 (F1~F10)
- **F1** 가중치를 ML 결과 보고 조정 = 순환 논증 (치명)
- **F2** r=0.42 ⇒ r²=0.18, "예측" 약함
- **F3** value_accuracy(reference 의존) 30% → 정의 모순
- F4~F10 가법성·일반화·모델·데이터셋·임계값·다지표·baseline

→ 두 층위 모두 처방 필요.

---

## 해결: v3.2 → v4 진화

### v3.2 (P1~P7 처방)
- `value_accuracy` 신설 (KS/TVD), outlier reference 고정
- 가중치 재배분, split-first 통합, 강도 0.9·0.95 추가
- 결과: r 0.085 → **0.420**

### v4 (F1~F10 처방, 정의 회복)
- value_accuracy 제거 → **label_consistency** + **feature_informativeness** 신설 (절대 품질 지표)
- 가중치 사전 등록 (ADR-009) + Hold-out 검증으로 순환 논증 회피
- 비선형 R²·min·임계값·다지표 분석 추가

→ **모두 split-first 원칙 준수, 단위 검증 스크립트로 사전 검증.**

## 결론
