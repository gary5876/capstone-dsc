# 캡스톤 주차 보고서 — Week 3 (줄글본)

- **작성일**: 2026-04-17
- **주제**: 데이터 품질 점수(DSC)와 머신러닝 모델 성능 간 상관관계 검증
- **보고 기간**: 2026-04-10 ~ 2026-04-17

---

본 캡스톤 프로젝트는 데이터 품질 점수 체계(DSC)가 실제 머신러닝 모델 성능과 상관관계를 가짐을 실험적으로 입증하는 것을 목표로 한다. DSC는 완전성, 유일성, 유효성, 일관성, 이상치 비율, 클래스 균형, 피처 상관 등 7개 지표의 가중 평균으로 데이터 품질을 0~100점으로 산출하는 체계이다.

이번 주에는 먼저 Google Drive와 Colab 기반의 실험 환경을 구축하고, 학술적으로 검증된 HPI 연구실의 오픈소스 DQ4AI를 오염 도구로 선정하였다. 이어서 규모와 타겟 유형이 다른 데이터셋 3종(TelcoCustomerChurn, SouthGermanCredit, Letter)과 모델 5종(LR, SVC, RF, XGBoost, MLP)으로 베이스라인을 측정한 결과, 원본 데이터 세 건 모두 A등급(97점대)으로 확인되어 "깨끗한 데이터 = 높은 DSC"라는 기준선을 확립하였다.

다음으로 5종의 오염을 4단계 강도(10~75%)로 주입한 결과, DSC 점수가 오염 강도에 충분히 반응하지 않는 결함을 발견하였다. 결측 placeholder를 감지하지 못하거나, 일관성 지표가 오염의 양을 반영하지 못하는 것이 주요 원인이었다. 이를 해결하기 위해 DSC 엔진 v2를 재설계하여 placeholder 감지, 오염 표식 행 비율 측정, 클래스 균형 민감도 개선을 적용하고, 전체 파이프라인을 재실행해 DSC 점수 59건과 모델 성능 295건을 확보하였다.

통계 분석 결과 Pearson 상관계수는 0.085(p=0.147)로 통계적 유의성은 확보하지 못하였으나, "DSC 10점 하락 시 F1 약 0.019 하락"이라는 방향성은 가설과 일치하였다. 75% 오염에서도 C·D 등급이 생성되지 않아 변화 폭이 부족했고, 숫자값 왜곡 오염을 탐지할 지표가 여전히 부재하였기 때문이다. 이는 가설의 반증이 아니라 실험 설계의 해상도 부족으로 해석된다.

차기 계획은 다음과 같다. 첫째, 분포 거리 기반 지표(KS 검정, Wasserstein)를 도입한 DSC 엔진 v3를 구현한다. 둘째, 오염 강도를 95%까지 확장하고 복합 오염을 도입해 C·D 등급 데이터를 확보한다. 셋째, 재실행을 통해 Pearson r ≥ 0.5, p < 0.01의 통계적 유의성 확보를 목표로 한다.
