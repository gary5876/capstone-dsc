## 2. 7개 문제점과 원인 진단

### P1. Outlier Ratio 지표가 가우시안 노이즈에 **역으로** 반응

**증거** — feature_accuracy 오염 강도별 outlier_ratio:

| dataset | lvl=0.0 | 0.1 | 0.25 | 0.5 | 0.75 | 추세 |
|---|---:|---:|---:|---:|---:|---|
| TelcoCustomerChurn | 1.0000 | 0.9997 | 0.9993 | 0.9969 | 0.9956 | 미세 하락 |
| **SouthGermanCredit** | 0.9450 | 0.9540 | 0.9670 | 0.9740 | **0.9873** | **상승** |
| **letter** | 0.9671 | 0.9772 | 0.9867 | 0.9906 | **0.9918** | **상승** |

**원인**:
- `calc_outlier_ratio`는 **현재 데이터의 IQR**로 outlier 판정 (`q1, q3 = s.quantile(0.25), s.quantile(0.75)`).
- `FeatureAccuracyPolluter._add_gaussian_noise_to_col` → `noise = N(0, level) * col_mean`을 가산. **분산이 커지면 IQR이 함께 넓어짐** → bound도 넓어짐 → outlier로 카운트되는 비율이 줄어듦.
- 결국 **노이즈가 강해질수록 "outlier가 적다"는 신호**가 발생, DSC 지표가 오히려 향상됨.

**파급**: 이 지표가 feature_accuracy 강도 0.75에서 letter의 outlier_ratio를 +2.5%p, SouthGerman을 +4.2%p 끌어올리며 DSC 총점을 베이스라인보다 **올림**(letter +1.58, SouthGerman +0.42). ML 성능 (letter F1 −0.42)과 정반대 방향.

**근본 결함**: 오염 후 데이터의 자기 IQR로 측정 → 변화 자체가 측정 기준에 흡수되는 자기참조 함정. 분포 거리 기반(베이스라인 IQR 고정 / KS test / Wasserstein) 지표가 필요.

---

### P2. Validity 지표가 "값"이 아닌 "타입"만 검사

**증거** — feature_accuracy 75%에서 validity가 데이터셋 3개 모두 1.0 유지.

**원인**:
- `calc_validity`: 수치형은 `pd.to_numeric(..., errors='coerce').notna()` 비율, 범주형은 길이 0~200 검사만.
- `FeatureAccuracyPolluter`는 수치형에 가우시안 노이즈를 가산(타입 유지), 범주형은 같은 컬럼 내 다른 카테고리로 swap(타입·도메인 모두 유지). → 어떤 검사도 걸리지 않음.

**파급**: 이 누락이 P1과 결합하여 **feature_accuracy 오염이 DSC 입장에서 "보이지 않는" 오염**이 됨. ML 모델은 강하게 영향(letter F1 0.957→0.46, Telco F1 0.74→0.42)을 받지만 DSC는 평온.

---

### P3. Consistency 지표가 ConsistentRepresentationPolluter 전용 시그니처(`-숫자` 접미사)에만 반응

**증거** — `calc_consistency`:
```python
has_suffix = s.apply(lambda x: bool(re.search(r'-\d+$', x)))
scores.append(1.0 - has_suffix.mean())
```

**원인**: 이는 ConsistentRepresentationPolluter가 새 representation을 만들 때 사용하는 명명 규칙(`'value' + '-' + 'i'`)을 직접 매칭. **실험 setup 안에서만 통하는 합성 신호**.

**파급**:
- 실험상 잘 작동(consistent_repr 폴루터에서 r=0.48~0.73)하지만, **실 운영 데이터의 "iPhone vs IPHONE vs i-phone" 같은 진짜 표현 불일치는 못 잡음**.
- 캡스톤 보고서에서 "DSC가 일관성을 측정한다"고 주장하기 어려운 구조.

---

### P4. Class Balance — 가중치 과소 + Letter처럼 다중클래스에서 무력

**증거**:
- 가중치: `'class_balance': 0.05` (전체 중 5%)
- letter (26개 클래스) class_balance 0.75: ΔDSC = 3.18 (전체 100점 중)
- letter F1 변화: ΔF1 평균 0.034, 최대 0.044 (RF 0.957→0.915)

**원인 두 갈래**:

1. **가중치 5% → 영향 천장이 5점**. 베이스라인 0.95 → 폴루션 후 0 으로 minratio가 떨어져도 점수 변화 ≤ 5점.

2. **`calc_class_balance`가 다중클래스에서 둔감**:
   ```python
   min_ratio = counts.min() / counts.sum()
   ideal_ratio = 1.0 / n_classes
   return min(min_ratio / ideal_ratio, 1.0)
   ```
   - 26개 클래스에서 `ideal_ratio = 1/26 ≈ 0.038`. min_ratio가 0.01까지 떨어져야 점수가 1.0→0.26으로 내려옴.
   - ClassBalancePolluter는 모든 클래스를 점진적으로 감소시키므로 letter min_ratio는 적당히 유지됨. 따라서 지표 변화가 미미.

**파급**: `letter × class_balance` 슬라이스의 상관계수 r=0.10 (사실상 0). letter는 상관관계 신호의 1/3을 차지하는데, 이 슬라이스가 통째로 노이즈가 됨.

---

### P5. Uniqueness 오염은 "DSC만 떨어지고 F1은 그대로"

**증거** — level=0.75에서:

| dataset | ΔDSC | ΔF1 평균 | ΔF1 최대 |
|---|---:|---:|---:|
| TelcoCustomerChurn | 15.00 | **0.0038** | 0.024 |
| SouthGermanCredit | 14.97 | 0.022 | 0.047 |
| **letter** | 14.80 | **−0.0036** (오히려 향상) | -0.001 |

**원인 (실험 설계 측면)**:
- UniquenessPolluter는 **train의 일부 행을 단순 복제**. 학습 모델 입장에서는 같은 샘플의 가중치가 1→k배가 되는 효과뿐, **새로운 정보가 추가되거나 손실되지 않음**.
- 트리 계열은 splitting 기준이 변하지 않고, 선형 계열은 동일 샘플의 loss 기여도만 비례 증가. 모두 일반화 성능에 거의 영향 없음.
- 반면 DSC `calc_uniqueness = 1 − (#duplicated/n)`은 row 수에 직격당해 큰 점수 하락.

**파급**: "DSC는 나쁘지만 ML은 멀쩡한" 케이스 60건이 통째로 발생 → 전체 상관계수의 분자를 0 방향으로 끌어내림. **개념적으로는 DSC가 옳고 ML이 둔감한 것**이지만, 캡스톤의 "DSC ↔ 성능" 가설은 실패한 것으로 보임.

---

### P6. DSC 측정 데이터 ≠ ML 학습 데이터 — 두 줄기 분리

**증거** — 노트북 02:
- Cell 9 (DSC 계산): `polluter.pollute(df_clean.copy())` → **전체 데이터**(예: Telco 7043행)에 오염 적용 후 점수 계산.
- Cell 12 (학습용 저장): `train_test_split` 후 `polluter.pollute(df_train_clean)` → **train(80%, 5634행)**에만 오염 적용 후 저장.

**원인**: split-first 안전 원칙(test 누수 방지)을 지킨 결과, **두 트랙이 서로 다른 데이터에 polluter를 새로 적용**. 동일 oeil seed라도 row 수·인덱스가 달라 결과 분포가 미세하게 다름.

**파급**:
- 결정론적 polluter(completeness, feature_accuracy)는 큰 영향 없음 — 점수는 수렴.
- 그러나 ClassBalancePolluter처럼 **샘플링이 들어간 polluter**는 train(80%)에서 만든 imbalance와 full에서 측정한 imbalance가 다를 수 있음.
- Uniqueness는 train에 복제된 비율이 full에 복제된 비율과 다를 수 있음 → DSC가 측정한 uniqueness 점수가 학습 데이터의 실제 uniqueness와 일치하지 않을 가능성.

**근본 결함**: **DSC는 학습용 train_polluted에서 측정해야 일관성**. "데이터셋 품질"의 정의가 흐려진 상태.

---

### P7. 베이스라인 자체에서 등급 차이가 거의 없어 등급 간 비교가 무의미

**증거** — 59건 DSC 점수 분포:
- A 등급 (≥90점): 50건 (84.7%)
- B 등급 (75-89): 9건 (15.3%)
- C/D 등급 (<75): **0건**

**원인**:
- 오염 강도 최댓값이 0.75에 머물고, completeness 외에는 75점 아래로 잘 안 내려옴.
- DSC 7개 지표 중 일부 지표만 폴루션에 반응하므로 가중평균이 자연스럽게 88+로 수렴.

**파급**: ANOVA로 "A vs B vs C" 비교를 못함(C/D 부재). 등급은 사실상 A·B 이진. 캡스톤 발표에서 "등급별 성능 차이"를 보여주기 어려운 데이터 분포.

## 4. 원인 분류 — 어디를 고쳐야 하나

| 문제 | 어느 층위 | 결함 종류 |
|---|---|---|
| P1. Outlier 역반응 | **DSC 엔진** | 측정 기준의 자기참조 |
| P2. Validity는 타입만 | **DSC 엔진** | 측정 항목 부재 (값 정확성) |
| P3. Consistency 시그니처 매칭 | **DSC 엔진** | 실험-과적합 (toy metric) |
| P4. Class balance 둔감 | **DSC 엔진** | 가중치 + 다중클래스 둔감성 |
| P5. Uniqueness vs ML 신호 차이 | **실험 가설** | DSC 가설과 ML 동작 간 본질 충돌 |
| P6. DSC 트랙 ≠ ML 트랙 | **실험 설계** | DSC 측정 대상 모호 |
| P7. 등급 분포 편향 | **실험 설계** | 강도 최댓값 한계 |
| S1~S2 | 로그 포맷 | 사소 |
| S3 | leakage 검증 | 정합성(현재는 안전) |
| S4 | 모델/전처리 | 평가 안정성 |

층위별 비중:
- **DSC 엔진 결함 (P1~P4)**: 핵심 폴루션 4종 중 3종을 미감지하거나 역으로 반응 → **상관관계가 깨지는 1차 원인**.
- **실험 설계 결함 (P6, P7, P5의 일부)**: DSC를 train에서 안 재고, 강도 0.75 천장에 등급 분포 편향 → **상관관계가 약하게 보이는 2차 원인**.
- **가설 자체 한계 (P5)**: uniqueness 같은 폴루션은 본질적으로 ML test에 영향이 적어, "DSC 만점에 가까운 데이터가 모델 성능 좋다" 명제가 항상 양방향이 아님.

## 5. 권장 후속 작업 (우선순위 제안)

1. **outlier_ratio 재설계** — 베이스라인 IQR 또는 z-score 분포를 고정해 "오염 후 데이터가 거기서 얼마나 벗어났는가"를 측정. 예: KS-distance, Wasserstein. → P1 해결.
2. **값 정확성(feature_accuracy) 지표 신설** — 가능하면 reference로부터의 분포 거리 기반. reference가 없으면 컬럼별 중앙치/IQR drift 측정. → P2 해결, feature_accuracy 슬라이스의 부호 반전을 양수로 회복.
3. **DSC를 train_polluted 위에서 측정** — Cell 9이 Cell 12 결과를 입력으로 받도록 통합. 한 번의 polluter 호출로 train만 오염시키고, 그 train으로 DSC와 ML을 모두 평가. → P6 해결.
4. **uniqueness 실험 처리 결정** — (a) 분석에서 제외하거나, (b) 가설을 "uniqueness는 정보 손실 없이 DSC만 떨어뜨리는 false-negative 사례"로 명시. → P5는 코드 수정이 아닌 해석 정정.
5. **오염 강도 0.9~1.0 추가 + class_balance 가중치 0.05→0.15 재조정** — 등급 C/D 케이스 확보. → P4·P7 해결.
6. **로그·SVC 안정화** — 음수 하락폭 표시 정리, SVC RBF kernel 시도. → S1·S4.

## 6. 보고서 한 줄 결론

> 캡스톤 가설("DSC ↔ ML 성능 상관관계")이 깨진 것은 **가설 자체가 틀렸기 때문이 아니라**, DSC 엔진이 5개 폴루션 중 3개(feature_accuracy, uniqueness, letter의 class_balance)에 대해 **무감각하거나 역방향으로 반응**하기 때문이다. completeness·consistent_repr·일부 class_balance 슬라이스에서는 r=0.5~0.8의 강한 양의 상관이 이미 입증되어 있으므로, **DSC 엔진의 P1·P2 두 결함만 고쳐도 전체 상관계수가 유의 수준으로 회복될 가능성이 높다**.
