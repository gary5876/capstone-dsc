## A. 용어집

### A.1 ML 일반

| 용어 | 정의 |
|---|---|
| **train / test split** | 데이터를 학습용·평가용으로 나누는 것. 보통 8:2. test는 학습 중 절대 안 봄. |
| **stratified split** | 분할 시 클래스 비율을 양쪽에서 동일하게 유지. 불균형 데이터에서 필수. |
| **leakage** | 학습 데이터의 정보가 평가 데이터로 새는 것. 결과가 비현실적으로 좋게 나와서 무효. |
| **F1 (macro)** | precision·recall의 조화평균. macro는 클래스별 F1을 단순 평균 (불균형의 영향 줄임). 0~1, 클수록 좋음. |
| **accuracy** | 맞춘 비율. 클래스 불균형이 심하면 신뢰 어려움. |
| **AUC-ROC** | 분류 모델이 양성·음성을 얼마나 잘 구분하는지. 0.5는 랜덤, 1은 완벽. |
| **R² (결정계수)** | 회귀 모델 성능. 1에 가까울수록 분산 설명 잘함. 0이면 평균 예측과 동급, 음수면 평균보다 못함. |
| **MAE / RMSE** | 회귀 오차. MAE = 평균 절대오차, RMSE = 제곱평균제곱근오차 (큰 오차에 더 패널티). |
| **cross-validation (CV)** | 데이터를 k등분해서 k번 돌려가며 학습·검증. 우연한 split의 영향 제거. |
| **분류 (classification)** | 타겟이 이산 카테고리인 task (예: 이탈/유지, A/B/C). |
| **회귀 (regression)** | 타겟이 연속값인 task (예: 주택 가격, 다음 달 매출). |

### A.2 이 프로젝트 고유 용어

| 용어 | 정의 |
|---|---|
| **DSC** | Data Score Card. 이 프로젝트가 만든 데이터 품질 점수 시스템. |
| **DSC Score** | 9개 metric 가중합 × 100. 0~100. |
| **DSC 등급** | A(90+), B(75-90), C(60-75), D(<60). |
| **DSC v2 / v3.2 / v4** | 엔진 버전. v2 → 1차 실패 (r=0.085), v3.2 → r=0.420 (단, 가중치 사후 조정 = 순환 논증으로 검증 정당성 의심), v4 → 정식 (r=0.598, 사전 등록 + hold-out 검증 통과). |
| **DSC v5** | 분류 cell만 있던 걸 회귀로 확장하는 framework 버전. 부품 단계, 미실행. |
| **cell** | (data_type, task) 쌍. 예: `(tabular, classification)` cell, `(tabular, regression)` cell. |
| **profile** | cell마다 지정되는 {metric_set, weights, evaluator}. `select_profile(data_type, task)` 가 반환. |
| **polluter** | 데이터에 표준화된 방식으로 오염을 주입하는 함수. 5종 사용. |
| **DQ4AI** | HPI 연구실의 polluter 라이브러리. 우리가 가져다 씀. |
| **baseline** | 오염 없는 원본 데이터셋. DSC와 모델 성능의 기준점. |
| **사전 등록 (pre-registration)** | 가중치·정의식을 실험 시작 전 ADR 문서에 고정. 결과 보고 변경 금지. 순환 논증 방지. |
| **순환 논증** | 가설이 두 변수의 관계를 묻는데, 한 변수(가중치)를 다른 변수(ML 성능)의 함수로 만들어 fit한 다음 "두 변수가 상관있다"고 주장하는 것. 두 변수가 독립이라야 그들의 상관 측정이 의미 있는데, 가중치 결정 절차에서 두 변수가 결합돼버림. ML 영역의 train-test contamination과 같은 카테고리. 본문 3.5절 F1 참조. |
| **polluter hold-out 검증** | 5개 polluter 각각을 빼고 r 측정. 어떤 polluter에 fitting됐다면 그걸 빼면 r이 무너져야 함. |

### A.3 통계 용어

| 용어 | 정의 |
|---|---|
| **Pearson r** | 두 연속 변수의 선형 상관 계수. −1 ~ +1. 0.3 이상이면 약한 상관, 0.5 이상이면 중간, 0.7 이상이면 강한 상관 (관행적 기준). |
| **r²** | Pearson r의 제곱. 한 변수가 다른 변수의 분산을 얼마나 설명하는지. r=0.6 → r²=0.36 (36% 설명). |
| **Spearman ρ** | 두 변수의 순위 상관. 비선형 단조 관계까지 잡음. Pearson과 달리 분포 가정이 약함. |
| **p-value** | 관측된 결과가 우연일 확률. 작을수록 우연이 아닐 확률이 높음. 0.05, 0.01, 0.001이 흔한 임계값. |
| **유의 (significant)** | p값이 임계값 이하라 우연으로 보기 어려운 수준. |
| **ANOVA** | 여러 그룹의 평균이 다른지 검정. F값 클수록·p값 작을수록 차이가 분명. |
| **분산 (variance)** | 값들이 평균으로부터 얼마나 흩어져 있는지. |
| **분포 (distribution)** | 값들의 빈도 패턴. |
| **KS distance** | 두 분포의 누적분포함수 차이의 최댓값. 0(같음) ~ 1(완전 다름). |
| **mutual information** | 두 변수가 얼마나 의존적인지. 한 변수를 알면 다른 변수의 불확실성이 얼마나 줄어드는지. 0이면 독립. |
| **entropy** | 분포의 불확실성. 균등 분포일수록 큼. |
| **IQR** | Interquartile Range. Q3 − Q1. 데이터 가운데 50% 너비. outlier 판정에 씀. |
| **KNN** | k-Nearest Neighbors. 한 점에서 가장 가까운 k개 이웃을 찾는 알고리즘. |
| **sparse (희소)** | 거의 모든 값이 0인 행렬·벡터 표현. onehot encoding 같은 걸 거치면 차원이 폭증하면서 sparse가 됨. 차원 수가 많은 sparse 데이터는 단순 모델로 잘 학습이 안 되는 경향. |
| **ablation (절제 분석)** | 시스템에서 한 부분을 빼고 다시 측정해서 그 부분의 기여도를 정량화하는 분석 방식. ML/SW 양쪽에서 흔히 씀. |
| **fitting** | 모델 파라미터를 데이터에 맞게 조정. 검증 맥락에서 "결과를 보고 가중치를 조정"한 걸 가리키면 부정직한 fit이라는 부정적 의미. |

## B. FAQ

### B.1 가설·방법론

**Q. 가중치를 결과 보고 정한 거 아닌가?**
A. v4의 가중치는 작업 시작 시점에 ADR-009에 사전 등록됨. 그 후 실험 결과를 봐도 가중치를 재조정하지 않음. 이걸 사후에 보장하기 위해 polluter hold-out 검증을 함 — 5개 polluter 중 어느 하나를 빼고 봐도 r이 유지되면 fitting이 아닌 일반화 신호. 5/5 모두 PASS.

**Q. r=0.598이 약한 거 아닌가?**
A. 절대값으로는 약함. 통계 관행으로 0.5~0.7은 "moderate". r²=0.358 → DSC가 ML 성능 분산의 36%만 선형 설명, 64% 미설명. 비선형 결합 R² 0.632 (63%). 그래서 주장 범위를 "예측"이 아니라 "유의한 양의 상관"으로 정직화. **DSC를 결정론적 예측 도구로 쓰면 안 됨** — 같은 등급 안에서도 F1 분산이 큼 (B등급 F1 0.52~0.97 범위). DSC가 의미 있게 쓰일 수 있는 방식은 (1) 데이터셋 간 상대 비교/랭킹, (2) 등급 간 그룹 비교, (3) 시간에 따른 모니터링. 본문 4.7 한계 5번 참조.

**Q. F1 macro를 메인 평가 지표로 쓴 이유는?**
A. 데이터셋이 클래스 불균형 (Telco 26.5% churn, Credit 30% bad, letter 26 다중 클래스). accuracy 단독은 majority class에 다 찍어도 점수 인플레됨. F1 macro는 precision·recall 조화평균을 클래스별 평균 → 작은 클래스도 동등 가중. 단 F1만 의존 안 함 — accuracy(r=0.50), AUC-ROC(r=0.46) 도 함께 측정해서 셋 다 양의 유의 상관 확인. 다지표 robustness 확보. 본문 2.8 참조.

**Q. 5개 모델로 일반화되는가?**
A. 알고리즘 5계열(선형 LR, 커널 SVC, 배깅 RF, 부스팅 XGB, 신경망 MLP)을 모두 커버. 모델별 r이 0.48 ~ 0.70 범위로 모두 양의 유의 상관.

**Q. 3개 데이터셋으로 일반화되는가?**
A. v4에서 3/3 모두 통계적 유의 (Telco r=0.29, SouthGerman r=0.23, letter r=0.80). letter가 압도적으로 높은 건 letter가 비교적 깨끗하고 구조가 단순한 다중 분류라 시그널이 강해서. Telco는 onehot 후 13,615차원 sparse라 단순 전처리에서 모델 학습 자체가 약함 (ADR-010에 데이터셋 한계로 인정).

### B.2 DSC 정의

**Q. 9개 metric 가중치는 어떻게 정했나?**
A. 사전 등록 시점에 도메인 휴리스틱으로 결정. completeness 0.20 (가장 기본), label_consistency 0.20 (학습 가능성 직접 측정), 나머지는 0.05~0.15. v3.2에서 v4로 갈 때 `value_accuracy 0.30` 슬롯을 `label_consistency 0.20 + feature_informativeness 0.10` 으로 분할 교체.

**Q. 왜 가중합인가? 비선형 결합이 더 좋다며?**
A. 비선형 R²이 1.77배 높지만 가중합을 유지. 이유는 **해석 가능성** — DSC의 정체성이 "각 차원 점수와 가중치를 보면 왜 이 점수가 나왔는지 설명 가능한 것". 비선형 결합은 ablation 분석으로만 보고. 가법성 한계는 정직 인정.

**Q. reference(원본 데이터)가 없으면 DSC 못 쓰나? (교수님 지적 사항)**
A. v3.2까지는 `value_accuracy` (weight 0.30) 가 baseline 분포 비교라 reference 필수였음 — 운영 환경에서 사용자가 raw 데이터만 가지면 사용 불가. v4에서 `value_accuracy` 제거하고 `label_consistency` (KNN), `feature_informativeness` (mutual info) 추가, 둘 다 데이터 자체에서 직접 계산. 결과적으로 **DSC v4 가중치 합 1.0 중 0.95는 reference 없이 작동**, 잔여 0.05만 `outlier_ratio` 가 baseline IQR 필요. 운용 옵션은 (1) 첫 데이터를 reference로 등록, (2) 도메인 IQR 사전 정의, (3) `outlier_ratio` 빼고 측정. 본문 3.6 (1) 참조.

### B.3 검증 결과 해석

**Q. `uniqueness` polluter에서 r이 약한 이유?**
A. uniqueness 폴루션은 train의 행을 단순 복제. 같은 행이 늘어나도 모델은 학습한 적 있는 데이터라 새로 배울 게 없음 → ML test 성능에 거의 영향 없음. DSC는 "데이터 다양성 감소" 신호를 보내는데 옳지만, 우리 ML 평가가 그 신호를 잡을 만큼 민감하지 않음. 이건 가설 반증이 아니라 평가 도구의 둔감함이라 한계로 인정.

**Q. ANOVA F=84.4가 무슨 의미?**
A. F값은 "그룹 간 분산 / 그룹 내 분산". 84.4는 매우 큼 — 등급(A/B/C/D) 간 평균 F1 차이가 등급 안의 변동에 비해 압도적으로 큼. p=5.79e-43은 사실상 0. 등급으로 ML 성능을 구분하는 게 통계적으로 신뢰 가능.

**Q. 합성 오염에서 자연 데이터로 일반화 가능?**
A. 본 연구는 합성 오염 한정 입증. 자연 발생 노이즈로 일반화는 후속 연구. 외부 타당성 입증하려면 자연 노이즈 데이터셋에서 재검증 필요.

**Q. 가우시안 노이즈가 자연계 노이즈를 다 cover하나?**
A. 아님. 가우시안은 N(0, σ²) 정규분포 노이즈로, 센서 측정 오차·신호 노이즈처럼 "독립 오차원의 합" 형태에는 잘 맞음 (중심극한정리). 하지만 salt-and-pepper noise (이미지에서 픽셀이 갑자기 0/255로 튐), dropout, heavy-tail 분포, 시스템적 bias 등은 가우시안으로 모델링 안 됨. `feature_accuracy` polluter는 가우시안만 다룸 → 한계 1번 (합성 오염 시나리오 한정) 의 일부.

**Q. 데이터셋 6개 (분류 3 + 회귀 3) 면 충분한가?**
A. framework 주장의 학계 관행 최소 조건은 충족. 다만 데이터셋 단위 n=3 이라 strong dataset 하나 (실제 letter r=0.80) 가 통합 r을 끌어올리는 영향 있음 (Telco r=0.29, Credit r=0.23). 도메인 일반화를 강하게 입증하려면 n을 더 늘려야 하지만 캡스톤 시간 제약으로 6개 결정. 텍스트·이미지·시계열 등 비정형 도메인은 out of scope, 후속 연구.

### B.4 v5 / 진행 상황

**Q. v5 회귀 cell 언제 끝나나?**
A. 마스터플랜 기준 Phase 1(인프라) 5~7일 + Phase 2(검증) 3~5일 + Phase 3(통합) 2~3일 ≈ 최소 10~15일. 옵션 Phase 4까지 가면 +1~2주. 캡스톤 발표 일정에 맞춰 결정.

**Q. 회귀 cell도 통과 못 하면?**
A. 분류 cell 결과는 그대로 유효. v5 framework 주장만 "회귀 cell 한계 인정 + 분류 cell 단독 주장"으로 후퇴. 가중치는 이미 사전 등록되어 있어서 결과 보고 fitting은 불가능 (정직성 유지).

**Q. cell이 2개여도 정말 framework인가?**
A. 학술 관행상 cell 2개는 framework 주장의 최소 조건. 더 강한 주장을 위해 옵션 Phase 4에서 clustering cell 추가하는 안이 있음. 캡스톤 범위에서는 2개로 충분, 후속 연구에서 더 추가하는 게 마스터플랜 7절의 in/out scope 정의.

**Q. 텍스트·이미지 데이터로의 확장은?**
A. Out of scope. 마스터플랜에 명시. 이유는 (1) 텍스트는 BERT급 모델이 필요해 학습 비용 큼, (2) 이미지·오디오는 polluter 자체가 다른 종류 (예: CIFAR-10-C는 weather corruption), (3) 캡스톤 시간 제약. 후속 연구 출발점.
