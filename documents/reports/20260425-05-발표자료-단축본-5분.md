## 슬라이드 2 — 1차 시도와 진단

### 1차 결과: 가설 입증 실패
**Pearson r = 0.085, p = 0.147 (비유의)** · 등급 A·B만 (C·D 0건)

### 그러나 슬라이스로 분해하면 부호가 정반대

| Polluter × Dataset | r |
|---|---:|
| feature_accuracy × **SouthGerman** | **−0.89** |
| feature_accuracy × **letter** | **−0.78** |
| feature_accuracy × Telco | +0.78 |
| completeness × 모든 데이터셋 | +0.53 ~ +0.64 |

→ **평균이 부호 반전을 가린다.** 가설이 틀린 게 아니라 **측정 도구의 결함** 의심

## 슬라이드 3 — 결정적 결함 P1·P2

![outlier 역반응](charts/20260425/04_outlier_p1_fix.png)

### P1. outlier_ratio가 가우시안 노이즈에 **역반응**
- 노이즈로 IQR이 함께 넓어져 "outlier가 줄어든 것"으로 측정 — **자기참조 함정**
- 빨간 선: v2에서 강도↑일수록 점수가 오히려 ↑

### P2. validity가 **타입만** 검사 → feature_accuracy 무감지
- 가우시안 노이즈는 type 보존 → validity 1.0 그대로
- 결과: letter feature_accuracy 75% → **DSC 1.58점 상승** (오염인데 점수↑)

---

## 슬라이드 4 — 해결: DSC 엔진 v3.2

### 핵심 변경 (8개 commit)
1. **`calc_value_accuracy` 신설** — 수치형 KS distance + 범주형 TVD (P2)
2. **`calc_outlier_ratio` reference 고정** — 베이스라인 IQR 기준 (P1)
3. **`calc_consistency` 재설계** — toy metric → reference 대비 새 표현 행 비율 (P3)
4. **가중치 재배분** — value_accuracy 0.30 신설, validity 축소
5. **노트북 02 split→폴루션→DSC 단일 트랙** — DSC와 ML 동일 데이터 (P6)
6. **오염 강도 0.9·0.95 추가** (P7) + leakage 검증 견고화 + SVC 안정화

→ 모두 split-first 원칙 준수, 8개 단위 검증 스크립트로 사전 검증

## 슬라이드 6 — 결론 + 교훈

### 캡스톤 가설 입증
**DSC 점수가 ML 성능을 통계적으로 예측한다 (r = 0.42, p ≈ 10⁻²⁰, n = 435)**

### 핵심 교훈 3가지
1. **가설이 안 맞으면 측정 도구를 먼저 의심하라** — 7개 결함 중 4개가 DSC 엔진 자체
2. **자기참조 함정** — 측정 기준이 측정 대상의 함수면 신호가 사라진다
3. **부호 반전을 평균이 가린다** — 슬라이스 분해가 진단의 시작

### 잔여 한계 (정직한 보고)
uniqueness 폴루션은 ML test에 본질적 영향 없음 → "DSC가 옳고 ML 평가가 둔감"한 사례. 가설의 반증이 아닌 평가 setup의 한계로 해석.
