# ADR-001: 실행 환경을 Google Drive + Colab으로 통일

- **일자**: 2026-04-10
- **상태**: 확정

## 배경

데이터셋 용량이 클 수 있고(최대 수백 MB~GB), 모델 학습에 GPU가 필요할 수 있으며, 로컬 PC의 저장 공간이 제한적이다.

## 선택지

| 선택지 | 장점 | 단점 |
|---|---|---|
| A. 로컬 PC에서 전부 실행 | 네트워크 불필요 | 저장 공간 부족, GPU 없음 |
| B. 로컬(코드) + Colab(학습) 분리 | 역할 분리 명확 | 데이터 업/다운로드 번거로움, Colab에서 로컬 .py import 불가 |
| **C. Google Drive + Colab 전부** | 데이터/코드/실행 한 곳, 세션 간 데이터 유지 | 네트워크 필수, Colab 세션 제한(12시간) |

## 결정

**선택지 C.** 모든 데이터, .ipynb, 결과를 Google Drive에 저장하고, Colab에서 실행한다. 로컬 PC에는 계획 문서(documents/)만 보관한다.

## 이유

- 로컬 PC 저장 공간 제약 해소
- Colab에서 Drive 마운트 한 줄로 전체 데이터 접근 가능
- .ipynb 파일도 Drive에 두면 Colab에서 바로 열 수 있어 별도 업로드 불필요
- sklearn 기반 모델이라 ~390회 실험이 3~4시간 내 완료 → Colab 세션 제한 문제 없음
- 처음에는 B안(로컬+Colab 분리)이었으나, 로컬 .py를 Colab에서 import할 수 없는 문제로 C안으로 변경
