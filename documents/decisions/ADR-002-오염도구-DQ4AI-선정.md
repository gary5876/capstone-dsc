# ADR-002: 데이터 오염 도구로 DQ4AI 선정

- **일자**: 2026-04-10
- **상태**: 확정

## 배경

데이터를 의도적으로 오염시켜 DSC 점수 변화와 모델 성능 변화의 상관관계를 입증해야 한다. 오염 도구가 필요하다.

## 선택지

| 선택지 | 설명 | 장점 | 단점 |
|---|---|---|---|
| **A. DQ4AI** | HPI 연구실, 6개 polluter 제공 | 학술 검증됨, 정형 데이터 전문, 우리 데이터셋과 호환 | 정형 데이터만 지원 |
| B. Metis | HPI 연구실, 품질 측정 프레임워크 | 6개 차원 측정 가능 | **오염(polluter) 기능 없음** — 측정만 가능 |
| C. 자체 구현 | 직접 polluter 작성 | 완전한 통제 | 시간 소모, 학술적 근거 부족 |
| D. DQ4AI(오염) + Metis(측정) | 둘 다 사용 | 오염 + 교차 검증 | 복잡도 증가 |

## 결정

**선택지 A. DQ4AI만 사용.** 여유가 되면 D(Metis 추가)로 확장.

## 이유

- Metis는 오염 기능이 없어서 DQ4AI를 대체할 수 없음
- DQ4AI의 metadata.json에 우리 3개 데이터셋이 이미 정의되어 있어 즉시 사용 가능
- DQ4AI의 polluter가 `compute_quality_measure()`도 제공 → DSC 지표 교차 검증에 활용 가능
- 학술 논문(2025, ScienceDirect)으로 검증된 방법론
