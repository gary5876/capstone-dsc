# DSC 검증 실험 — 전체 진행 보고서

> **목적**: DSC(Data Score Card) 점수가 실제 ML 모델 성능과 상관관계가 있음을 입증
> **기간**: 2026-04-10 ~ 진행 중

---

## 1. 프로젝트 설계 (2026-04-10)

### 1.1 핵심 아이디어
```
깨끗한 데이터 → DSC 점수 높음 → 모델 성능 좋음
오염된 데이터 → DSC 점수 낮음 → 모델 성능 나쁨
∴ DSC 점수 변화량 ≈ 모델 성능 변화량 → 상관관계 입증
```

### 1.2 실험 파이프라인
```
Phase 1: 환경 세팅 + 데이터 다운로드
Phase 2: 베이스라인 DSC 점수 + 모델 성능 측정
Phase 3: DQ4AI로 데이터 오염 생성 + 오염 DSC 점수 측정
Phase 4: 오염 데이터로 모델 재학습 + 성능 측정
Phase 5: 결과 통합 + 시각화 (차트 5종)
Phase 6: 통계 검증 (피어슨/스피어만 상관계수)
```

### 1.3 주요 설계 결정
| 결정 | 내용 | 근거 |
|---|---|---|
| 실행 환경 | Google Drive + Colab | 로컬 저장 공간 부족, Colab에서 .py import 불가 (ADR-001) |
| 오염 도구 | DQ4AI (HPI 오픈소스) | 학술 검증됨, 데이터셋 호환, 품질 측정 포함 (ADR-002) |
| 데이터셋 | Telco, Credit, letter 3개 | DQ4AI 호환 + 크기/타겟/피처 다양성 (ADR-003) |
| 모델 | LR, SVC, RF, XGB, MLP 5개 | 계열별 대표 + 오염 민감도 스펙트럼 (ADR-004) |
| Consistency | LLM 없이 처리 | DQ4AI quality_measure 활용 (ADR-005) |

### 1.4 실험 규모
```
데이터셋 3개 × polluter 5개 × 강도 4단계 × 모델 5개 = 300회 학습
+ 베이스라인 3 × 5 = 15회
= 총 315회 이상
```

### 2.4 DSC 스코어링 엔진 구현
7개 품질 지표를 계산하는 엔진 구현:

| 지표 | 가중치 | 계산 방법 |
|---|---|---|
| Completeness | 0.25 | 1 - (null / total) |
| Uniqueness | 0.20 | 1 - (duplicated / n) |
| Validity | 0.20 | 수치형 변환 성공률 |
| Consistency | 0.15 | 범주값 변형 표현 비율 |
| Outlier Ratio | 0.10 | 1 - (IQR 이상치 / n) |
| Class Balance | 0.05 | 엔트로피 / 최대 엔트로피 |
| Feature Correlation | 0.05 | 1 - (고상관 쌍 비율) |

종합: `DSC Score = Σ(지표 × 가중치) × 100`, 등급: A(90+) / B(75~89) / C(60~74) / D(<60)

## 3. Phase 3: 오염 생성 & DSC 점수 측정 (2026-04-12)

### 3.1 오염 실행
DQ4AI의 5개 polluter × 4개 강도(10/25/50/75%) × 3개 데이터셋 = 60건 시도

| Polluter | 오염 방식 | 대응 DSC 지표 |
|---|---|---|
| CompletenessPolluter | 결측값(placeholder) 주입 | Completeness |
| UniquenessPolluter | 행 복제 | Uniqueness |
| FeatureAccuracyPolluter | 수치: 가우시안 노이즈, 범주: 스왑 | Validity |
| ConsistentRepresentationPolluter | 범주값에 접미사 추가 | Consistency |
| ClassBalancePolluter | 클래스 비율 왜곡 | Class Balance |

### 3.2 에러 발생: FeatureAccuracyPolluter × TelcoCustomerChurn (ADR-007)
**증상**: 4개 강도 모두에서 `ValueError: Could not convert string to numeric`
**원인**: TotalCharges 컬럼에 공백 문자열 `" "` 존재 → pandas가 object 타입으로 인식 → polluter가 문자열 연결 수행
**대응**: polluter 적용 전 `pd.to_numeric(errors='coerce').fillna(0)` 전처리 추가

### 3.3 핵심 문제 발견: DSC 점수가 오염에 반응하지 않음 (ADR-008)

**증상** (TelcoCustomerChurn 기준):

| 오염 유형 | 10% | 25% | 50% | 75% | 변화폭 |
|---|---|---|---|---|---|
| completeness | 99.16 | 99.03 | 98.65 | 97.20 | -1.97 |
| uniqueness | 92.51 | 89.17 | 85.84 | 84.17 | -15.00 |
| feature_accuracy | ERROR | ERROR | ERROR | ERROR | - |
| consistent_repr | 92.14 | 92.14 | 92.14 | 92.14 | 변화 없음 |
| class_balance | 99.94 | 99.77 | 99.06 | 97.72 | -1.45 |

**75% 오염을 가해도 최저 점수가 84점(B등급)**. C, D 등급 데이터가 전혀 없음.

**원인 분석**:

1. **Completeness 지표**: CompletenessPolluter는 `-1`이나 `"empty"` 같은 placeholder를 삽입하는데, DSC 엔진은 `isnull()`만 카운트 → placeholder를 결측으로 인식 못함
2. **Consistency 지표**: 오염 강도를 바꿔도 점수가 92.14로 고정 → 지표 계산 로직이 오염의 "양"을 반영하지 못함
3. **Class Balance 지표**: 엔트로피 기반이라 클래스 비율이 상당히 왜곡되어야 변화 → 민감도 부족

**프로젝트에 미치는 영향**:
- DSC 점수 분포가 84~99에 밀집 → 산점도에서 상관관계 보이지 않음
- C/D 등급 없음 → 등급별 성능 비교 불가
- **핵심 주장 "DSC 하락 → 성능 하락" 입증 불가**

- `data/polluted/` — 오염 데이터 파일들
