### 2. 발견한 에러: FeatureAccuracyPolluter × TelcoCustomerChurn
**증상**: `Could not convert string '29.851889.5108.15...' to numeric`

**원인 분석**:
- TelcoCustomerChurn의 `TotalCharges` 컬럼에 공백 문자열 `" "`이 포함 (11건)
- pandas가 해당 컬럼을 object(문자열) 타입으로 인식
- FeatureAccuracyPolluter가 수치형이 아닌 문자열로 판단 → 가우시안 노이즈 대신 **문자열 연결(concatenation)** 수행
- 결과: `"29.85" + "1889.5" + "108.15" + ...` → 수만 글자의 비정상 문자열 생성

**대응**: `pd.to_numeric(errors='coerce').fillna(0)` 전처리 추가

### 3. 핵심 문제 발견: DSC 점수가 오염에 반응하지 않음

오염 강도를 75%까지 올려도 DSC 점수가 거의 변하지 않는 심각한 문제 발견:

| 오염 유형 | 10% | 25% | 50% | 75% | 문제 |
|---|---|---|---|---|---|
| completeness | 99.16 | 99.03 | 98.65 | 97.20 | **75% 오염인데 3점만 하락** |
| uniqueness | 92.51 | 89.17 | 85.84 | 84.17 | 유일하게 의미 있는 변화 |
| feature_accuracy | ERROR | ERROR | ERROR | ERROR | Telco에서 전부 실패 |
| consistent_repr | 92.14 | 92.14 | 92.14 | 92.14 | **강도 무관하게 동일** |
| class_balance | 99.94 | 99.77 | 99.06 | 97.72 | 변화 미미 |

(위 표는 TelcoCustomerChurn 기준, SouthGermanCredit/letter도 유사한 경향)

**원인 분석**:

#### Completeness 지표 문제
- CompletenessPolluter는 결측값으로 **placeholder**를 삽입 (수치형: `-1`, 범주형: `"empty"`)
- DSC 엔진의 `calc_completeness()`는 `df.isnull().sum()`으로 계산
- **placeholder(-1, "empty")는 null이 아니므로 감지 불가** → 점수 하락 없음

#### Consistency 지표 문제
- ConsistentRepresentationPolluter는 범주값에 `-1`, `-2` 접미사 추가 (예: `"Yes"` → `"Yes-1"`)
- DSC 엔진의 `calc_consistency()`는 `re.sub(r'-\d+$', '', x)`로 접미사 제거 후 비교
- 그런데 오염 강도(`percentage_polluted_rows`)를 바꿔도 점수가 **항상 92.14**로 동일
- 추정: polluter의 강도 파라미터가 의도대로 동작하지 않거나, 지표 계산 로직에 버그

#### Class Balance 지표 문제
- 엔트로피 기반이라 클래스 비율이 상당히 왜곡되어야 점수가 떨어짐
- ClassBalancePolluter의 `imbalance_level=0.75`에서도 2점 정도만 하락

### 4. 이 문제가 프로젝트에 미치는 영향

DSC 점수가 오염에 반응하지 않으면:
- **차트 1 (산점도)**: DSC 점수가 84~99 구간에 밀집 → 상관관계 보이지 않음
- **차트 4 (등급 박스플롯)**: C, D 등급 데이터 없음 → 등급별 비교 불가
- **통계 검증**: 피어슨 상관계수가 약하게 나올 가능성 높음
- **핵심 주장 불가**: "DSC 점수 하락 → 모델 성능 하락" 증명 실패
