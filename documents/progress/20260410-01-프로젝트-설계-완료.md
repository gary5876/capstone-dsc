# 2026-04-10: DSC 검증 실험 프로젝트 설계 완료

## 작업 내용

### 1. 마스터플랜 수립
- DSC 검증 실험의 전체 파이프라인 설계
- 핵심 아이디어: 깨끗한 데이터(DSC 높음, 성능 좋음) vs 오염 데이터(DSC 낮음, 성능 나쁨) 상관관계 입증
- 6단계 Phase로 구성

### 2. 실행 환경 확정
- 초기안: 로컬(코드) + Colab(학습) 분리
- 변경: **전부 Google Drive + Colab** (로컬 PC에는 문서만)
- 이유: 로컬 저장 공간 제약, Colab에서 로컬 .py import 불가

### 3. 데이터셋 선정
- DQ4AI metadata.json 분석 → 호환 데이터셋 파악
- **TelcoCustomerChurn, SouthGermanCredit, letter** 3개 확정
- 크기/타겟/피처 다양성 확보

### 4. 모델 선정
- DQ4AI 논문의 11개 모델 분석
- **LR, SVC, RF, XGB, MLP** 5개 확정 (계열별 대표)
- 오염 민감도 스펙트럼: 민감(LR, SVC) ~ 강건(RF, XGB) ~ 중간(MLP)
- 선정 이유 5가지 기준으로 문서화

### 5. DQ4AI Polluter 분석
- 6개 polluter의 소스코드 분석 완료
- 오염 강도: 0~100% (5% 단위, 21단계) → 우리는 5단계(0/10/25/50/75%) 선택
- Polluter ↔ DSC 지표 매핑 확정

### 6. 비정형 데이터 확장 설계
- 텍스트/이미지/음성에 대한 오염 방법, DSC 지표, 모델, 데이터셋 설계
- 일정/실험 수 등 현실 라인 없이 "무엇을 해야 하는가"만 정리

### 7. 문서 체계 구축
- documents/ 아래 plans, progress, decisions 구조
- README.md: 용어 정의 + 문서 구조 안내
- ADR 5건 작성 (환경, 오염도구, 데이터셋, 모델, Consistency)
